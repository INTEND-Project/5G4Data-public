Intent Report Simulator

Welcome to the intent report simulator for the EU funded INTEND project. The simulator is developed as part of the 5G4Data use-case and allows you to create intent report simulators. A simulator in this setting is replacing how a TM Forum intent handler would report observations related to TM Forum intents issued by an intent owner. We are using version 3.6.0 of the TM Forum Intent ontology (TIO).

Create live simulators

Generate observation reports and store the reports in GraphDB or Prometheus. You can configure how reports are generated in many ways (random, from file, at a given frequency, within a given range, etc).

Populate database with reports

Create many (historical) intent reports in one operation and store them in a GraphDB or Prometheus. This is a great way to populate the storage with lots of reports for the intents already added to the GraphDB intents_and_intent_reports knowledge graph. Before using this option you need to use the intent simulator to generate intents in GraphDB. You will be able to configure how the reports should be created (frequency, value range, different value generation modes and anomaly modes).
